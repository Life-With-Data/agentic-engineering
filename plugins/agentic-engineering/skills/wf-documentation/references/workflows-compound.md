# Compound completed engineering work

Use this reference at the workflow boundary after a change has shipped or the
repository's delivery process otherwise considers it complete. It coordinates
durable knowledge capture; it does not choose the repository's documentation
layout, knowledge tools, tracker, or publication mechanism.

## Entry gate

Proceed only when:

- the engineering outcome is complete according to `wf-delivery`;
- the result produced a reusable lesson that is not already maintained;
- the mapped `documentation` capability identifies a durable owner.

If the work is still under review or delivery, return to the owning workflow.
If repository documentation guidance is missing, route to `wf-setup` rather
than inventing a location or tool.

## Coordinate the record

1. Read the change, its verification evidence, and the mapped documentation
   targets.
2. Use [compound docs](compound-docs.md) to decide what should be retained.
3. When useful, gather problem context, root cause, solution, verification, and
   prevention notes independently. Research helpers return evidence; they do
   not choose repository policy or write competing files.
4. Amend the existing durable owner when one exists. Create a new document only
   in a location and format established by the repository.
5. Run the repository's documentation checks and verify links and examples.
6. Deliver the change through the repository's mapped documentation or delivery
   path.

## Optional integrations

Repositories may map search indexes, knowledge graphs, decision logs, or memory
systems through their documentation guidance. Refresh those systems only when
the mapped asset explicitly requires it. Their absence is not a workflow
failure, and this plugin does not install, configure, or infer one.

## Completion

Report the durable source, its owning repository capability, validation
evidence, and any follow-up that could not be completed. The workflow is done
when the lesson is findable through the repository's established paths, not
merely when a file has been written.
