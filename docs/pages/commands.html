Command Reference

Here's the thing about slash commands: they're workflows you'd spend 20 minutes doing manually, compressed into one line. Type /plan and watch three agents launch in parallel to research your codebase while you grab coffee. That's the point—automation that actually saves time, not busywork dressed up as productivity.

Workflow Commands (8)

/workflows:brainstorm

Explore requirements and approaches through collaborative dialogue before planning implementation

/workflows:brainstorm
/workflows:compound

Document a recently solved problem to compound your team's knowledge

/workflows:compound
/workflows:groom

Groom intake into ready work — drive an idea, bug report, or stub issue through brainstorm → plan, then STOP once the item is planned on the board with a join-keyed plan doc and sub-issues. Never implements; the deliverable is a groomed, ready-to-claim work item.

/workflows:groom
/workflows:merge

Merge a PR — a thin entry point to the land-pr skill (CI wait, review-thread resolution, merge gate, branch cleanup, idempotent tracker-item close).

/workflows:merge
/workflows:orchestrate

Drive the brainstorm → plan → work → review → land → compound pipeline as the orchestrator — delegating implementation to sub-agents, reviewing their work, and running fully autonomously by default (surfacing only genuine blockers; --final-review adds one pre-merge check). Segment flags bifurcate the run — --groom stops once the item is planned; --implement starts from groomed work and refuses to groom.

/workflows:orchestrate
/workflows:plan

Transform feature descriptions into well-structured project plans following conventions. Idempotent entry gate — re-running on an already-planned item routes to work instead of re-planning; orchestrate invokes this directly.

/workflows:plan
/workflows:review

Perform exhaustive code reviews using multi-agent analysis and worktrees

/workflows:review
/workflows:work

Execute work plans efficiently while maintaining quality and finishing features

/workflows:work
